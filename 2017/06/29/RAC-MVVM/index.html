怎样使用RAC+MVVM

发表于 2017-06-29   |   分类于 iOS   |     |   阅读次数

　　自从接触了RAC+MVVM后，后续的项目基本全是使用这种模式来写的，算是接触很久了，今天就分享下其在项目中的使用。
　　讲MVVM之前，我们先来说说MVC

MVC

　　
　　Model：模型，封装、存储、处理数据
　　View：视图，负责UI展示
　　Controller：控制器，负责逻辑处理，协调Model和View
　　
　　
　　随着需求越来越多，功能越来越复杂，如果我们将逻辑处理、网络请求全放在Controller来处理，Controller里势必会越来越臃肿，最后变成Massive View Controller(MVC)。这时我们就需要寻找另外一种模式来代替MVC，于是MVVM进入了我们的视野。 　　

MVVM

　　
　　Model：封装、存储、处理数据
　　View：负责UI展示
　　ViewModel：负责业务逻辑处理，将MVC的业务逻辑抽离出来放在此处
