上架到App Store时，如何规避审查

发表于 2017-09-27   |   分类于 iOS   |     |   阅读次数

　　在知乎上看过这样的一个段子，大意是这样：如何评价iOS和安卓？有大神答，安卓：这个权限给不给，不给就不给你用；iOS：我能获取你这个权限吗?
　　做iOS的同学都会遇到app审核不通过的情况，可能有人说，我就没有遇到过，那说明你的iOS生涯是不完整的，iOS生涯怎么能缺少苹果审核团队给你发的审核不通过的通知邮件呢。
　　
　　相对于安卓，iOS上的流氓软件极少，一部分原因是iOS是闭环的，还有一部分原因就是苹果的审核机制，只要违反规定，就会审核不通过，严重时更为做下架处理。可是实际开发中，难免会遇到可能会触碰到规定的时候，审核不通过，直接影响到app上线的时间，这时就要规避审查，下面讲讲如何规避审查。
　　
　　iOS app都会有个版本号，例如正在销售的app版本号为1.1.0，现在要上架的版本号为1.2.0，这时我们就可以利用版本号来避开某个操作，因为苹果审核人员在审核你的app时，你的正在销售的app版本仍为1.1.0，只有审核通过了，才会变成1.2.0，所以我们可以通过判断正在销售的app的版本和正在审核的版本是否相同，来暂时屏蔽掉某个触碰规定的操作。
